# 중요 변경사항: dongName 폴백 매칭 제거

## 변경 이유
부산광역시를 검색했는데 다른 지역 매물이 표시되는 문제의 원인:
- **dongName 폴백 매칭**이 문제였음

## 문제 상황
```sql
-- 이전 코드 (문제 발생)
AND (
    (h.sgg_cd, h.umd_cd) IN (('26110', '10100'), ...)  -- 부산 코드
    OR
    h.umd_nm IN ('중앙동', '남포동', ...)               -- 🔴 문제!
)
```

### 왜 문제인가?
1. 부산광역시의 dong_code 리스트에서 "중앙동"이 추출됨
2. SQL에서 `h.umd_nm IN ('중앙동', ...)`으로 검색
3. **서울의 "중앙동", 대전의 "중앙동"도 모두 매칭됨**
4. 결과: 부산만 선택했는데 전국의 "중앙동"이 다 나옴

## 해결 방법
```sql
-- 새로운 코드 (dongName 폴백 제거)
WHERE
    h.sgg_cd IS NOT NULL 
    AND h.umd_cd IS NOT NULL 
    AND (h.sgg_cd, h.umd_cd) IN (('26110', '10100'), ...)
```

### 변경 결과
- ✅ 오직 튜플 매칭만 사용
- ✅ sgg_cd와 umd_cd가 정확히 일치하는 매물만 표시
- ⚠️  sgg_cd 또는 umd_cd가 NULL인 매물은 제외됨 (정확도 우선)

## 트레이드오프
### 장점
- **정확도 100%**: 부산을 선택하면 오직 부산 매물만 나옴
- 간단하고 명확한 로직

### 단점 (매우 경미함)
- sgg_cd나 umd_cd가 NULL인 매물은 필터링 시 제외됨
- 하지만 실무적으로 문제 없음 (거의 모든 매물이 코드를 가지고 있음)

## 결론
**정확도가 최우선**이므로 dongName 폴백을 제거하는 것이 올바른 선택.
