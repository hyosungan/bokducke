# dong_code 슬라이싱 논리 검증

## 가정
- dong_code는 10자리 문자열 (예: "2611010100")
- sgg_cd는 앞 5자리 (예: "26110")
- umd_cd는 뒤 5자리 (예: "10100")
- houseinfos 테이블에 sgg_cd와 umd_cd가 분리되어 저장됨

## 검증 필요 사항

### 1. dong_code 길이 확인
- dong_code가 항상 10자리인가?
- 9자리나 11자리인 경우가 있는가?

### 2. sgg_cd와 umd_cd 길이 확인
- sgg_cd가 항상 5자리인가?
- umd_cd가 항상 5자리인가?
- NULL이거나 빈 문자열인 경우가 있는가?

### 3. 일관성 확인
- CONCAT(sgg_cd, umd_cd) = dong_code가 항상 성립하는가?
- 일부 데이터에서 불일치가 있는가?

### 4. 실제 데이터 확인 필요
실제 데이터베이스에서 샘플을 확인해야 함.
